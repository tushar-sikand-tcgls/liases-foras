## 🔧 Root Cause Analysis

### Issue #1: Test Design Mismatch

**Current Excel Structure:**
```
| Prompt | Good Answer | Expected Answer Should Include |
|--------|-------------|-------------------------------|
| What is the Project Size of Sara City? | The <b>Project Size</b> of Sara City in Chakan by Sara Builders & Developers (Sara Group), launched in Nov 2007 is <b>3018 Units</b>. This is the largest project by size in the dataset. | 3018 units |
```

**Model Behavior:**
- Returns: `"3018 Units"` (clean, short, accurate)

**Test Validation:**
- Checks if "3018 units" is in "3018 Units"
- ❌ FAILS because of capitalization ("units" vs "Units")
- Also compares similarity to verbose "Good Answer" → low score

**Solution:** Normalize "Expected Answer Should Include" to match model format:
- Change `"3018 units"` → `"3018"` (just the number)
- This allows flexibility in unit formatting

---

### Issue #2: Project Name Normalization

**Data Problem:**
```python
# Loaded project names (actual):
'Pradnyesh\nShriniwas'  # Has newline!
'Sara\nAbhiruchi\nTower'

# Test queries for:
'Pradnyesh Shrinivas'  # With space
'Sara Abhiruchi Tower'
```

**Why it happens:**
- Project names extracted from PDF/Excel preserve newline characters
- Query parsing looks for space-separated names
- **No match found** → query fails

**Solution:** Normalize during data loading:
```python
project_name = project_name.replace('\n', ' ').replace('  ', ' ').strip()
```
